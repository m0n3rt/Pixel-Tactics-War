# 藏品配置使用说明

本文说明如何维护商店藏品配置，以及何时使用导出工具函数。

## 1. 文件位置

- 藏品配置文件：`relics.json`
- 游戏逻辑文件：`main.js`
- 导出工具函数：`exportRelicDefs()`（定义在 `main.js` 中）

## 2. 日常修改（推荐）

绝大多数情况下，直接改 `relics.json` 即可。

### 步骤

1. 打开 `relics.json`
2. 修改目标藏品字段（如 `price`、`desc`、`maxStacks`、`rarity`）
3. 保存文件
4. 刷新游戏页面

完成后会自动读取新配置，不需要手动复制代码。

## 3. 配置字段说明

每个藏品对象建议包含以下字段：

- `key`: 唯一键（需与代码逻辑一致）
- `name`: 藏品名
- `price`: 金币价格
- `maxStacks`: 最大叠层
- `rarity`: 稀有度（如 `rare` / `epic` / `legendary`）
- `desc`: 前端展示描述

示例：

```json
"belt": {
  "key": "belt",
  "name": "腰带",
  "price": 3,
  "maxStacks": 1,
  "rarity": "rare",
  "desc": "+2%步兵生命值"
}
```

## 4. 何时使用 exportRelicDefs

仅在你先改了 `main.js` 里的 `RELIC_DEFS`，想把当前内存定义导出回 JSON 时使用。

### 使用步骤

1. 运行游戏页面
2. 打开浏览器开发者工具 Console
3. 输入并执行：

```js
exportRelicDefs()
```

4. 函数会：
- 在控制台输出格式化 JSON
- 尝试自动复制到剪贴板
5. 将输出内容粘贴覆盖到 `relics.json` 并保存

## 5. 常见问题

### Q1：我只改了 `relics.json`，还要执行 `exportRelicDefs()` 吗？

不需要。改完 JSON 后刷新页面即可生效。

### Q2：我改完没生效怎么办？

按顺序检查：

1. `relics.json` 是否是合法 JSON（逗号、引号、括号是否正确）
2. 页面是否已刷新
3. 浏览器控制台是否有 `relics.json` 加载失败提示

### Q3：如果 `relics.json` 加载失败会怎样？

游戏会退回到 `main.js` 中内置默认 `RELIC_DEFS`，仍可运行，但不会使用你刚改的 JSON。

## 6. 建议

- 平时只维护 `relics.json`
- 尽量不要同时在 JSON 与 `main.js` 双改同一字段，避免混淆
- 大改动后，建议进商店开几次宝箱做快速验证（价格、描述、可购买性是否符合预期）

## 7. 如何新增藏品

新增藏品目前是“JSON + 少量代码挂接”的方式，按下面清单操作即可。

### 7.1 最小步骤清单

1. 在 `relics.json` 新增藏品对象（至少含 `key/name/price/maxStacks/rarity/desc`）。
2. 在 `main.js` 的 `playerRelics` 初始对象补上该 `key`（需要同步三处：全局初始、`newGame()`、`backToTitle()`）。
3. 如果该藏品要进入宝箱随机池，在 `CHEST_RELIC_WEIGHTS` 中加入该 `key` 和权重。
4. 在 `mkUnit()` 或对应逻辑里实现藏品效果（例如生命/攻击/攻速/移速等）。
5. （可选）在 `buildStatsHTML()` 里补展示文案，方便在“增益总览”中看到实际效果。
6. 若它属于“集齐后解锁胜利象征”的基础藏品，再把它加入 `BASE_RELIC_KEYS`；否则不要加。

### 7.2 JSON 示例

```json
"newRelicKey": {
  "key": "newRelicKey",
  "name": "新藏品名字",
  "price": 6,
  "maxStacks": 1,
  "rarity": "epic",
  "desc": "效果描述"
}
```

如果需要“只改 JSON 就生效”，请给藏品加上 `effects` 字段（本项目已支持效果解释器）。

```json
"newRelicKey": {
  "key": "newRelicKey",
  "name": "新藏品名字",
  "price": 6,
  "maxStacks": 1,
  "rarity": "epic",
  "desc": "效果描述",
  "effects": [
    { "type": "atkFlat", "value": 1 },
    { "type": "hpMul", "value": 0.05 }
  ]
}
```

### 7.3 effects 字段支持类型（可直接配）

- `atkFlat`: 攻击力固定值加成（可配 `unitType`）
- `rangeFlat`: 射程固定值加成（可配 `unitType`）
- `hpMul`: 生命百分比加成（如 `0.1` 表示 +10%，可配 `unitType`）
- `atkMul`: 攻击百分比加成（可配 `unitType`）
- `moveSpeedMul`: 移速百分比加成（通过减少移动CD实现，可配 `unitType`）
- `atkSpeedMul`: 攻速百分比加成（通过减少攻击CD实现，可配 `unitType`）
- `dpCapFlat`: 部署点上限固定加成
- `unitCapFlat`: 单位上限固定加成
- `globalAllStatsMul`: 全属性百分比加成（用于类似“胜利的象征”）
- `archerExtraArrowChance`: 弓手追加箭概率（可叠层）
- `battleGoldFlat`: 每场结算金币固定加成
- `atkSpeedByGold`: 按金币计算攻速（需配 `goldStep`、`stepBonus`、`cap`）
- `moveSpeedByGold`: 按金币计算移速（需配 `goldStep`、`stepBonus`、`cap`）

说明：

- `unitType` 可选值：`infantry` / `archer` / `tank` / `cavalry`。
- 默认按层数叠加；若要关闭叠层可配置 `"perStack": false`。

### 7.4 常见漏改点

- 只改了 `relics.json`，但忘了在 `playerRelics` 三处补 key，导致无法正确计层。
- 加了藏品定义，但没加 `CHEST_RELIC_WEIGHTS`，导致宝箱永远刷不到。
- 有描述但没实现效果逻辑，导致“看起来有，实战不生效”。

### 7.5 新增后快速自测

1. 进游戏并进入商店，确认新藏品可出现、价格正确、可购买。
2. 购买后打开增益总览，确认层数与说明一致。
3. 进入战斗验证效果是否真实生效（例如面板数值、战斗体感、日志/表现）。
